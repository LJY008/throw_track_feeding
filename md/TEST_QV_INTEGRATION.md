# 测试QV集成功能

## 快速测试步骤

### 1. 启动程序
```bash
cd e:\nc-code
python neck_throat_track.py
```

### 2. 期待现象

#### 启动阶段
- ✅ 看到 "✅ QV实时检测绘图窗口已创建"
- 出现两个窗口：
  - **主窗口**: Neck Throat Tracking（原有的Y/Z位移图）
  - **新窗口**: QV实时检测监控（3子图）

#### 运行阶段
1. **数据采集**（前10秒）：
   - 控制台每10帧打印：`[数据点] 30点, 吞咽Y=...mm, 吞咽Z=...mm`
   - QV窗口显示：子图1和子图2实时更新Y/Z位移曲线

2. **首次检测**（10秒后）：
   - 控制台输出：
     ```
     组内最优检测 | 抑制窗口=1.0s | H=1.5 H2=-2.0
     ```
   - 如果有吞咽动作：
     ```
     🎯 检测到 1 个事件:
        事件 1: 时间=3.456s (索引=103)
     ```
   - 子图3显示：组合信号曲线 + 红色星号标记事件 + 黄色高亮区域

3. **持续检测**（每隔1秒）：
   - 自动运行检测
   - 实时更新事件计数

### 3. 验证功能

#### 测试1：吞咽检测
**操作**: 做一个明显的吞咽动作
**预期**: 
- 子图3出现红色星号
- 控制台显示 "🎯 检测到 X 个事件"
- 窗口标题更新总事件数

#### 测试2：身体运动抑制
**操作**: 前后移动身体
**预期**:
- 控制台显示 "🚶 [身体运动触发] 斜率变化..."
- Y/Z信号置零（子图1和2显示平直线段）
- 不产生误检事件

#### 测试3：实时更新
**预期**:
- 曲线持续更新（每帧）
- 窗口不卡顿
- 延迟 < 100ms

### 4. 性能指标

| 指标 | 目标值 | 验证方法 |
|------|--------|----------|
| 检测延迟 | < 1秒 | 吞咽后1秒内看到标记 |
| 缓冲时间 | 10秒 | 启动后10秒开始检测 |
| 检测间隔 | 1秒 | 连续吞咽至少间隔1秒 |
| 刷新率 | ~30fps | 曲线流畅无卡顿 |

### 5. 常见问题

#### 问题：QV窗口未出现
**检查**:
```bash
# 查看配置文件
cat tracker_config.json | grep qv_enabled
# 应该显示: "qv_enabled": true
```

#### 问题：无检测输出
**原因1**: 数据不足（< 10秒）
- 等待至少10秒

**原因2**: 信号幅度太小
- 做更明显的吞咽动作
- 检查Y/Z位移曲线是否有变化

**原因3**: 阈值过高
- 临时降低阈值测试：
  ```json
  "qv_volatility_threshold": 1.0  // 从1.5降到1.0
  ```

#### 问题：误检太多
**解决**:
```json
{
  "qv_volatility_threshold": 2.0,  // 提高阈值
  "slope_stable_threshold": 0.05,  // 更敏感的身体运动检测
  "qv_suppress_slow_motion": true  // 确保启用
}
```

## 对比测试

如需与原qv_realtime_monitor对比：

### 旧方案（ZMQ通信）
```bash
# Terminal 1
python neck_throat_track.py

# Terminal 2
python qv_realtime_monitor.py
```

### 新方案（集成）
```bash
# 只需一个命令
python neck_throat_track.py
```

**优势**:
- 启动更简单（一个命令 vs 两个命令）
- 延迟更低（<10ms vs ~100ms）
- 更稳定（无网络通信故障）
- 资源占用更少（单进程 vs 双进程）
