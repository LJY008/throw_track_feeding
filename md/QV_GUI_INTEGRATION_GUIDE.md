**数据流**:

### 操作流程

1. **启动程序**
   - OpenCV窗口显示相机画面
   - Tkinter窗口显示QV监测GUI

2. **自动采集数据**
   - 程序自动追踪颈部和喉部
   - 数据实时传递到QV GUI
   - 文件标签显示：`实时数据: XXX点, XX.X秒`

3. **实时检测**
   - 数据达到100点后自动开始检测

   - 红色虚线：最终选中的最优事件

## 事件标记样式说明

### 子图4：检测结果

```
信号曲线
    ▲
    │     ╔════════╗         ╔═══════╗
    │     ║ 组1    ║         ║ 组2   ║  ← 蓝色半透明框（事件组区间）
    │     ║ 1.2s   ║         ║ 0.8s  ║     标注：组号 + 持续时间
    │ ____╝____|____╚_________╝___|___╚____
    │          ┊                  ┊
    │          ┊ 红色虚线         ┊ 红色虚线  ← 最优事件（组内评分最高）
    │          ┊                  ┊
    └──────────┴──────────────────┴──────► 时间(秒)
               3.5s              7.8s
```

**说明**:
- **蓝色框**：抑制窗口内的所有候选事件作为一个"事件组"
- **红色虚线**：从该组中选出的最佳事件（基于波动率+二阶导数+位移变化的综合评分）

### 组内最优选择

算法在每个事件组内：
1. 筛选候选事件（波动率>=H 且 二阶导数<=H2）
2. 验证位移变化（必须>=min_displacement_change）
3. 评分：`score = w1×波动率 + w2×二阶导数 + w3×位移变化`
4. 选择评分最高的候选作为最终事件

## 参数调优指南

### 提高检出率
```
h: 0.3-0.4（更敏感）
H: 1.0-1.2（降低阈值）
min_displacement_change: 0.5（降低位移要求）
```

### 减少误检
```
h: 0.6-0.8（更平滑）
H: 2.0-3.0（提高阈值）
H2: -3.0至-5.0（更严格的二阶导数要求）
suppression_window: 1.5-2.0（扩大抑制窗口）
```

### 双轴vs单轴
- **双轴（推荐）**: 综合Y和Z方向，更鲁棒
- **单轴**: 仅用Z方向，计算更快
- **Z轴取反**: 双轴模式下，将Z轴反向后与Y轴平均

## 故障排查

### 问题1: 导入错误
```
❌ 无法导入QV监测GUI: No module named 'qv_realtime_monitor'
```
**原因**: qv_realtime_monitor.py不在同目录  
**解决**: 确保两个文件在同一目录（e:\nc-code）

### 问题2: 窗口不显示
**原因**: matplotlib后端问题  
**解决**: 检查qv_realtime_monitor.py开头是否有 `matplotlib.use('TkAgg')`

### 问题3: 数据不更新
**检查**:
- 文件标签是否显示"实时数据: XXX点"
- 数据点数是否>=100
- "实时更新检测"是否勾选

### 问题4: 参数修改无效
**解决**: 勾选左侧的"实时更新检测"复选框
